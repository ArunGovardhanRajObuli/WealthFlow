# Documents Controller

> [!info] File Upload Gateway
> The **documentsController.js** manages the HTTP layer for metadata validation before allowing files to hit the file system.

## Core Responsibilities
1. **Category Enforcing**: Restricts uploaded documents to an explicit allowlist (`['identity', 'financial', 'medical', 'property', 'insurance', 'other']`), falling back to `other` if anomalous data is injected.
2. **File Propagation**: Passes the structured metadata object along with the actual `req.file` (generated by multer middleware in the router) down to the `documentService.createDocument` function.
3. **Empty File Blocking**: If the user tries to create a document without attaching a file, it catches the specific "No file uploaded" error and cleanly maps it to a 400 Bad Request.
